Architecture de l'application

Notre application répartie est composée des éléments suivants :

  • Frontend : Interface web utilisant HTML, CSS, JavaScript et Leaflet pour afficher une carte interactive centrée sur Nancy.
  • Backend : Service RMI déployé sur les machines de l'IUT, interrogeant une base de données Oracle pour les données des restaurants.
  • Base de données : Hébergée sur charlemagne.iutnc.univ-lorraine.fr:1521/infodb, contient les tables RESTAURANTS, TABLES_RESTO et RESERVATIONS.
  • Proxy : Service HTTP (à l'aide de HttpServer) pour contourner les restrictions CORS et accéder à des données bloquées.
  • API externes : Utilisation de l'API Cyclocity (https://api.cyclocity.fr) pour les données des stations Vélo, l'API data.education.gouv.fr pour afficher les établissements scolaires...

Technologies utilisées

  • Frontend : HTML, CSS, JavaScript, Leaflet 1.9.4
  • Backend : Java (RMI, HttpClient, HttpServer), JSON pour les échanges de données.
  • Base de données : Oracle (via JDBC).
  • Hébergement : Serveur webetu pour le site, machines de l'IUT pour les services RMI.
  • Versionnement : Git, hébergé sur GitHub.

Ce qui a été réalisé

  • Carte interactive centrée sur Nancy avec trois couches de base : OpenStreetMap, OpenStreetMap.HOT, OpenTopoMap.
  • Affichage des stations Vélo avec leurs adresses et capacités, des établissements scolaires, des restaurants et des incidents de la circulation.
  • Interface utilisateur avec navigation entre la carte et la page de compte rendu.
  • Interactions RMI avec un serveur central (topologie en étoile)
  • Réservations de tables dans les restaurants via une interface dédiée.
  • Utilisation d'un proxy pour contourner les restrictions CORS et accéder aux données des API externes.
  • Déploiement sur le serveur webetu et les machines de l'IUT.
  • Gestion des erreurs et des exceptions pour une meilleure robustesse de l'application.

Instructions d'utilisation

Pour utiliser l'application :

  1. Déployez les services sur les machines via le script de déploiement.
  2. Accédez au site via le serveur webetu.
  3. Cliquez sur l'icône de configuration en bas à gauche de la page pour renseigner l'adresse IP du service central.
  4. Les différentes données se chargent alors automatiquement sur la page.
  5. Utilisez le contrôleur de couches (en haut à droite) pour sélectionner les données à afficher et pour changer la couche de base (Nancy Map, Nancy Map HOT, Nancy Topo Map).
