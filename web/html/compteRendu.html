Architecture de l'application

Notre application répartie est composée des éléments suivants :

  • Frontend : Interface web utilisant HTML, CSS, JavaScript et Leaflet pour afficher une carte interactive centrée sur Nancy.
  • Backend : Service RMI déployé sur le serveur de l'IUT, interrogeant une base de données Oracle pour les données des restaurants.
  • Base de données : Hébergée sur charlemagne.iutnc.univ-lorraine.fr:1521/infodb, contient les tables RESTAURANTS, TABLES_RESTO et RESERVATIONS.
  • Proxy : Service HTTP (à l'aide de HttpServer) pour contourner les restrictions CORS et accéder à des données bloquées.
  • API externes : Utilisation de l'API Cyclocity (https://api.cyclocity.fr) pour les données des stations Vélo.

Technologies utilisées

  • Frontend : HTML, CSS, JavaScript, Leaflet 1.9.4
  • Backend : Java (RMI, HttpClient, HttpServer), JSON pour les échanges de données.
  • Base de données : Oracle (via JDBC).
  • Hébergement : Serveur webetu pour le site, machines de l'IUT pour les services RMI.
  • Versionnement : Git, hébergé sur GitHub.

Ce qui a été réalisé

  • Carte interactive centrée sur Nancy avec trois couches de base : OpenStreetMap, OpenStreetMap.HOT, OpenTopoMap.
  • Affichage des stations Vélo avec leurs adresses et capacités, récupérées via l'API Cyclocity.
  • Interface utilisateur avec navigation entre la carte et la page de compte rendu.

Instructions d'utilisation

Pour utiliser l'application :

  1. Accédez au site via le serveur webetu.
  2. La carte affiche par défaut les stations Vélo et les restaurants .
  3. Utilisez le contrôleur de couches (en haut à droite) pour activer/désactiver les stations Vélo ou les restaurants, et pour changer la couche de base (Nancy Map, Nancy Map HOT, Nancy Topo Map).
